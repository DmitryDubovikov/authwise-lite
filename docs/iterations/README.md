# docs/iterations — формат

Каждая закрытая итерация NN получает каталог `docs/iterations/NN/` из трёх документов:

- **`explainer.md`** — простым языком, что итерация делает и зачем. Обязательные секции:
  «Зачем это (продукт и ценность)» (первой), «🧵 Что это дало резюме» (якорь красной нити),
  «TL;DR (простыми словами)», мини-ликбез «Что это за техника» (для не-каркасных итераций),
  «Поток данных» с ASCII-схемой и таблицей «Инструмент | Что делает | Куда пишет»,
  «Карта „где в коде“» — нумерованный список с цитатами кода.
- **`demo.md`** — happy-path смок-чеклист: пронумерованные шаги с копипаст-исполнимыми
  командами и ожидаемым результатом под каждым. Шаги с live LLM помечаются
  «⚠️ live, стоит денег» и в церемонию закрытия не гоняются.
- **`learnings.md`** — решения и почему, грабли, отложенные TODO и осознанные срезы
  (помеченные `# aw-lite:` в коде).

Доки пишутся живой прозой (полными предложениями), без плейсхолдеров: каждая команда
исполнима как есть на этой машине. Номера строк в «Карте» — ориентир на момент итерации;
надёжнее искать по именам символов.
