## Рубрика

**CRITICAL — сломан шов конституции** (компаундится между итерациями, ведёт к болезненному рефактору):
- `domain/` импортирует что-либо из `app/*` (нарушено чистое ядро).
- Workflow импортирует транспорт (`cli`); поток не строго `transport → workflow → domain/persistence`.
- Запись в MLflow (golden-сет, версии промптов, policy-версии, aliases) **вне boundary**; MLflow-хендл дотянулся до `domain` или workflow-ноды.
- **Ассерты golden/CI сверяются не с `PathTrace` из domain, а с телеметрией** (OTel-спаны как источник истины пути) — правило 6.
- Чтение env через `os.environ`/`os.getenv` мимо `app/config.py` (`Settings`).
- LLM-вызов приложения **не через `route()`** (прямой `openai`/`litellm` SDK в модуле); нарушен единый chokepoint.
- **Нарушена дисциплина LiteLLM (правило 5, красная линия):** использован LiteLLM **Proxy** вместо SDK; добавлены callbacks/success_callback; включена телеметрия; версия не запиннена; ключ/base_url мимо `Settings`.
- Тест или path-gate, дёргающий `route()`, не в `replay` по умолчанию / путь, способный уйти в живой вызов без кассеты (в т.ч. CI-джоб гейта).
- Закоммичен секрет (`.env` с реальными ключами, хардкод ключа).
- Разрастание домена сверх toy (правило 3): новые ветки/ноды графа, новые поля `PathTrace` сверх `{branch, retry_cycles, nodes}`, новые поля golden-записи, доля джокеров в golden-сете >20% (размывает гейт).

**BUG — нарушен контракт:** `route()` отдаёт не единый тип; граф-раннер возвращает ответ без `PathTrace` (или `PathTrace`, не согласованный с фактически пройденными нодами); неверный async, ведущий к дедлоку на boundary; конфликт alias/версии в MLflow с моделью «промпты отдельно + policy-версия пинит».

**NIT — отклонение от аналога/именования**, судейский вызов. Только с конкретным аналогом.

**Срез без маркера (NIT).** Сознательный угол (упрощение, заглушка, «одного хватит») без комментария `aw-lite:` — отметь как NIT с предложением добавить маркер `# aw-lite: <потолок> → <апгрейд>`. Срез *с* маркером — не находка: это штатный режим проекта, не недоделка. Не превращай маркер в повод докрутить срез до «по-взрослому» — потолок назван осознанно (правило 1).

## Что НЕ флагаешь (existence-gate — иначе шум)

Существо PA-решений (правильно ли «по-медицински» одобрять/отклонять) / реализм текстов заявок · качество промптов · edge-cases toy-домена · прод-хардненинг (auth/rate-limit/CORS/retries/perf) · CI/Dependabot · security сверх утечки секретов и дисциплины LiteLLM · широта тестов. Всё это проект осознанно режет (см. CLAUDE.md «Что осознанно НЕ делаем»).

**Но красную линию (правило 1) — флагаешь.** Existence-gate режет реализм данных и точность, НЕ: корректность демонстрируемой техники, швы, утечку секретов, дисциплину LiteLLM (это уже в CRITICAL выше), **и продвижение красной нити** (north star). Отдельные кейсы-нарушения правила 1, а не штатный срез (CRITICAL/BUG по тяжести):
- **path-gate, который на деле не блокирует** (регрессия маршрутизации не валит CI; membership-ассерт написан так, что проходит любой путь) — корректность демонстрируемой техники.
- **golden-сет, размытый до беззубости** (широкие `allowed_paths` у большинства записей — гейту нечего различать; квота ≥80% singleton нарушена).
- **итерация не двигает заявленный пункт красной нити** — резюме-ключевик не стал демонстрируемым.
- **реализация отрезает финальную витрину** (правило 8: напр. Langfuse подключён голым OTel — Agent Graph недоступен).
