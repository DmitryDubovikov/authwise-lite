# Итерация 00 — решения и грабли

## Решения и почему

- **Авторские кассеты вместо record-прогона.** ROADMAP планирует record-расход только на
  iters 1/2/5, поэтому smoke-кассеты сгенерены скриптом: содержимое строится domain-схемами
  (`Classification`/`PolicyCheckResult`) и потому валидно по построению, `usage` синтетический,
  но формат боевой — с ним уже можно писать код cost-атрибуции. Помечено в коде
  `# aw-lite: авторские кассеты → реальный record в iter 1`.
- **Порт MLflow — 5051, не 5050.** У сиблинга triagewise-lite MLflow уже держит 5050 на этой
  машине, а сиблинги должны жить параллельно. Зафиксировано в `docker-compose.yml`, дефолте
  `Settings.mlflow_tracking_uri` и `.env.example`.
- **Тиры перенесены из triagewise как есть** (решение с пользователем 2026-07-06): те же три
  снапшота, консистентность семьи важнее свежести моделей. Новое против triagewise — цены
  токенов лежат в самом `llm-tiers.yaml` (сквозной контракт №5), поэтому формат файла стал
  структурным (`model` + `input_per_1m`/`output_per_1m`), и `load_tiers` парсит его схемой.
- **`route()` возвращает `LLMReply{content, usage}`, а не голую строку.** Кассетный формат
  обязан хранить `usage` уже с iter 0, и глупо было бы тут же его выбрасывать на выходе шва:
  iter 2 (`RunRecord`) заберёт usage без смены сигнатуры.
- **Номер попытки policy-check не хранится в стейте** — он выводим: `retry_cycles + 1`
  (находка /simplify). Меньше полей — меньше способов разъехаться.
- **MLflow python-клиент в зависимости не добавлен** — в iter 0 он не используется (есть только
  контейнер); появится в iter 1 вместе с Evaluation Dataset и пином `>=3.4`.
- **GitHub-репо — публичный** (решение с пользователем 2026-07-06): как у сиблингов, плюс
  branch protection для iter 2 на приватном бесплатном плане недоступна.

## Грабли

- **Сиблинги делят локальные порты.** «Перенос compose 1-в-1» из triagewise молча наследует и
  его порт — контейнер не стартовал (`Bind for 0.0.0.0:5050 failed`), пока сиблинг жив. Урок
  на будущие итерации: Langfuse/Phoenix/Prometheus/Grafana тоже проверять на коллизии портов
  с triagewise (у него Phoenix уже на 6006).
- **Кассетные форматы сиблингов различаются существенно.** policywise хранит только текст ответа
  (без `usage`) — перенеси мы его формат, per-node cost в iter 3–4 было бы не из чего считать.
  Сработала сверка обоих аналогов перед переносом: формат взят из triagewise, ключ — свой
  (контракт №4).

## Отложенные TODO / осознанные срезы

- **Авторские кассеты без `messages`** — провенанс появится при первом настоящем record (iter 1).
- **Рендер пути собирается из `(branch, retry_cycles)`, а не из `nodes`** — при замороженном
  графе это эквивалентно и проще; если граф когда-нибудь разморозят (не планируется), рендер
  придётся пересобрать по `nodes`.
- **CI зелёный только после первого пуша** — Actions-скелет лежит в дереве, но прогнать его до
  первого `git push` физически нечем; проверка бейджа — на пользователе после коммита.
